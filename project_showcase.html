Project Deep Dive

Real-Time Weather Monitoring Pipeline

Executive Summary

This document details the design and implementation of an automated, cloud-based data pipeline. The system automatically fetches real-time weather data from a public API, transforms it into a clean and structured format, and loads it into a relational database. Built on Microsoft Azure, the pipeline runs on a high-frequency schedule (every 3 minutes), providing a reliable stream of weather data without manual intervention, demonstrating a modern serverless ETL process.

Project Goal

To build a robust, automated pipeline that collects, cleans, and stores live weather data for a specific location (Mississauga, ON), simulating a real-world requirement for a logistics or transportation company.

Key Features

  • Fully automated and event-driven
  • High-frequency updates (every 3 minutes)
  • Built on a serverless cloud architecture
  • Transforms raw JSON into structured SQL data

Implementation Steps

The solution was built in three distinct phases: setting up the cloud infrastructure, defining the data transformation logic, and orchestrating the automated pipeline.

Prerequisites & Infrastructure

  • Resource Group: A logical container named `WeatherProject-RG` was created to hold all project-related resources.
  • Azure SQL Database: A new SQL Database was provisioned. A `CREATE TABLE` script was executed to create the destination table `dbo.WeatherLog`.
  • Azure Blob Storage: A Storage Account was created with a container named `raw-weather-data` to act as the landing zone.
  • Azure Data Factory: An ADF instance was created as the development and execution environment.

Transformation Logic

A Data Flow named `weatherprojectdataflow` performs the core data cleansing:

  1. Source: Reads raw JSON files from the Blob Storage container.
  2. Select (`select1`): Renames nested JSON fields (e.g., `current.temperature_2m` to `TemperatureCelsius`) and removes unneeded columns.
  3. Derived Column (`added2columns`): Enriches the data by adding `City` and `PipelineRunTime`, and crucially, fixing the `WeatherCode` data type from boolean to integer using the expression `iif(WeatherCode, 1, 0)`.
  4. Sink (`outputdatabase`): Writes the final, clean data to the `dbo.WeatherLog` table with manual mapping for schema correctness.

Orchestration & Automation

A pipeline named `weatherproject` was created to manage the sequence of operations and automate the entire process.

  • Copy Data Activity: Performs the "Extract". It fetches data from the Open-Meteo API and drops the raw JSON file into Blob Storage.
  • Data Flow Activity: Performs the "Transform" and "Load". It runs the `weatherprojectdataflow` on the raw file produced by the previous step.
  • Schedule Trigger: The automation engine. It was configured to run the entire pipeline automatically every 3 minutes.

Pipeline Execution Flow

When triggered, the pipeline follows a precise sequence of events. Press Play to see it in action.

Step 1: The Schedule Trigger fires.
Step 2: The Copy Data activity initiates a `GET` request to the Open-Meteo API.
Step 3: The API returns a raw JSON response, saved as a file in Azure Blob Storage.
Step 4: Upon success, the Data Flow activity begins.
Step 5: The Data Flow reads the new JSON file from Blob Storage.
Step 6: It executes all transformations: selecting, renaming, and fixing data types.
Step 7: The final, structured row is loaded into the **`dbo.WeatherLog` table** in Azure SQL Database.
Step 8: The pipeline run is complete and waits for the next scheduled trigger.

Conclusion and Business Value

This project successfully demonstrates the creation of an end-to-end, automated data pipeline using Azure Data Factory's visual tools. The final solution is robust, serverless, and requires no manual maintenance.

The business value of such a pipeline is immense. It transforms raw, unstructured data into a valuable, queryable asset. This clean data can be directly connected to analytics dashboards (like Power BI) for live monitoring or used by data scientists to build predictive models, enabling data-driven decision-making for the organization.
